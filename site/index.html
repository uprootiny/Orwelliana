Trace

Every action is typed and attributable.

Events carry channel, event, summary, details, and confidence, so a run can be replayed and audited without reading the code first.

Conversation

History is stored as operator-grade state, not prompt sludge.

Sessions, roles, and bounded context windows are explicit. You can reuse the latest safe slice without letting prompt growth become a hidden failure mode.

Deploy Doctor

Infrastructure prerequisites are checked before redeploy.

The deploy doctor inspects Git state, recent CI and Pages runs, Pages provisioning, and public reachability so platform failures stop looking like app failures.

Why It Exists

Most agent logs are output. This is state.

The point is not pretty telemetry. The point is making operator judgment inspectable: what the agent saw, what it changed, what it believed, what failed, and which slice of conversation is still safe to feed back into the loop.

Pipeline

Pure model, explicit effects

1 Observe trace, repo, fleet
2 Bound conversation window
3 Verify CI, Pages, reachability
4 Act deploy, health-check, iterate

Runbook

Technique encoded

  1. Prove the local state is coherent.
  2. Check VCS state and branch drift.
  3. Inspect provider state before assuming code broke deploy.
  4. Verify public reachability after the platform says green.
